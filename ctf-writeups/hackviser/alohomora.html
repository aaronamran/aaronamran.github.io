Git is a version control system that supports multi-user collaboration by tracking code changes. The version control history of projects is stored in the ".git" folder. This writeup focuses on practicing directory discovery in a web application, detecting critical data by navigating between git branches, and gaining access to the server with a key file.

Tasks

Considering that the chances of obtaining email address information via terminal is low, we can try to directly open the target IP in a web browser and look for contact information. We are greeted with the following:

Clicking on the About section and reading further leads us to the following:

Answer: tommy@cyberwand-blog.com

Question 2: What is the name of the directory that contains git-related files found during a directory scan?

To approach this, we have to use directory discovery techniques to find hidden directories. By using the command gobuster dir -u 172.20.4.90 -w /usr/share/seclists/Discovery/Web-Content/common.txt -t 50, we found a directory named ".git" which contains git-related files.

Answer: .git

Question 3: What is the username of the developer?

Opening .git in the web browser reveals the following git directory structure:

We can use git-dumper to download the git folder to our local machine and search for information easily.

Now that we have the git repository, we will use the git command line tool to access various information needed of the repository.

After navigating to the target directory and using the command git log, we can find the commit that contains the username of the developer.

Answer: tomriddlex1

Question 4: Which branch is active?

Simply use the command git branch to see the active branch.

Answer: main

Question 5: What is the git command showing commits?

Answer: log

Question 6: What is the git command that changes the branch?

To change the branch, we can use the command git checkout <branch_name>.

Answer: checkout

Question 7: What is the name of the forgotten file in the dev branch?

We need to switch the branch from main to dev using the command git checkout dev. Then use ls -l to view available files.

Answer: id_rsa

Question 8: What is the password hash for user hackviser?

To access the password hash, we need to access the system and access /etc/shadow. We can use the id_rsa file as a key to access the system via SSH. However, the permissions for the id_rsa file need to be set correctly using chmod 600 id_rsa, and then we can attempt SSH again.

After successfully accessing the system, we can use cat /etc/shadow to view the password hash for user hackviser.

Answer: $y$j9T$FOWx5qCAorpq72xggPErc0$zkgSTMnKfdrb/jH1zRKBvHCIsNCtmPElDaM4TjhNE7B
